removeIf的底层讲解：

interface Predicate<T>{
  boolean test(T t);
}
interface Collection {
  public default boolean removeIf(Predicate<String> predicate){
    Iterator<String> it = this.iterator();
    boolean b = false;
    while (it.hasNext()) {
      String next =  it.next();
      if (predicate.test(next)){
        it.remove();
        b = true;
      }
    }
    return b;
  }
}

增强for和普通for的比较：

int[] arr = new int[3];
for (int a : arr) {  // a不是索引 而是元素值
  System.out.println(a);
}

for (int i = 0; i < arr.length; i++) {  //i 是索引
  System.out.println(arr[i]);
}

// 增强for中 是不会出现索引的 （底层毕竟是迭代器  迭代器没有索引）
// 普通for中 是有索引的

class Student implements Iterable<String>{  // iterator 迭代   iter 简写形式  able 可以....的   Iterable  可以迭代的。
  String name;
  String sex;
  public Student(String name, String sex){
    this.name = name;
    this.sex = sex;
  }

  @Override
  public Iterator<String> iterator() {
    return new Iterator<String>() {
      @Override
      public boolean hasNext() {
        return true;
      }

      @Override
      public String next() {
        return name;

数据结构： 数据在容器中排列方式。

栈 ： 先进后出 队列： 先进先出 数组： 查询快 增删慢。 链表 ： 查询慢 增删快。 单向链表 ： 只能从前往后查 节约一些空间 双向链表： 既可以往前查找 也可以往后去查 ， 浪费一些空间。 数据的存储设计： 拿时间换空间， 拿空间换时间。

数据结构设计的精髓 ： 时间和空间的转换。 一个储藏室里面 存储产品： 平铺： 浪费空间，但是查找速度特别快 放架子： 平衡一点了 时间也比较快， 空间也能充分利用一点 堆满： 空间利用满了， 查找速度特别慢了。

举例： 链表的 单向链表 和双向链表。

题目的思想 ：拿时间换空间， 拿空间换时间 题： 有一个容器， 里面存储了1万个 连续的数 0-9999 (数的顺序是打乱的) ， 但是这些数组里面 其中一个数 是缺失的， 有一个数 是重复的。 比如： 0 1 2 3 7 5 6 7 8 9 – 7 是重复的 4是缺失的。 顺序乱的： 9 6 7 3 2 7 5 1 0 8 – 7 是重复的 4是缺失的。 请你 用最快速的方法， 找到 那个缺失的数， 找到那个重复的数。

List 的儿子：

ArrayList ：底层是数组结构 底层数组初始长度是10, 每次扩容1.5倍

LinkedList ：链表结构的方法 特有方法：头和尾相关的方法 因为 LinkedList 底层是双向链表。 操作里面的元素， 都要从头往后 或者从后往前找。

public void addFirst(E e)	在该列表开头插入指定的元素
						public void addLast(E e)	将指定的元素追加到此列表的末尾
						public E getFirst()		返回此列表中的第一个元素
						public E getLast()		返回此列表中的最后一个元素
						public E removeFirst()		从此列表中删除并返回第一个元素
						public E removeLast()		从此列表中删除并返回最后一个元素

数据结构中 只有数组有索引， 双向链表是没有索引的， 也就是说 LinkedList的底层是没有索引的 但是 LinkedList 是 List 接口的子类 那么他就具备 List 的那些方法啊。

List 的特有方法：
  void add(int index , E e)
  E remove(int index);
E set(int index , E e)
  E get(int index )
  那些方法 都是通过索引来操作元素的， 然而 LinkedList 底层却没有索引 岂不矛盾？？

我大概给你说一下 LinkedList 底层是怎么 模拟索引的。

public E get(int index ){
  int count = 0; //计数器
  int size0 = this.size(); // size0 记录了 当前LinkedList集合有多少个元素。
  if (index < size0/2){ // 从前往后
    while (true){
      //从前往后找的代码
      //每找一次
      count++;
      if (count == index){
        //找到了
        return 那个值 ;
      }
    }
  }else{
    while (true){
      // 从后往前找
      //每找一次
      size0--;
      if (size0==index){
        //找到了
        return 那个值;
      }
    }

  }

}

LinkedList 的源码分析：大致理解就可以了

​ 1：LinkedList 底层是用的双向链表 ​ 2：LinkedList 存储进去的元素， 他给你把元素封装为了 Node 对象。（Node 对象中 封装三个属性 当前值 上一个元素的地址 下一个元素的地址值） ​ 3：LinkedList 只能从头往后 或者从后往前找 但是他却提供了一个 get(int index) 的方法。

LinkedList<String> ll = new LinkedList<>();
ll.add("a");
ll.add("b");
ll.add("c");

String str = ll.get(1);  // 这岂不是也能通过索引获取元素吗？

// 为什么LinkedList 提供了一个get方法呢。  因为他实现了List接口 ， 而List接口中有一个get方法。
//  但是 LinkedList 的get方法底层 绝对不是通过索引拿的，
// 而是先去判断 你拿的位置是属于 链表的前半部分 还是后半部分。
// 如果属于前半部分 就会从第一个往后找， 如果属于后半部分 就会从最后一个往前找。
