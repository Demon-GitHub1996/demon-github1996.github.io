io总结

io流结构

结论:

​ 输入流是读文件的

输出流是写文件的

1.字节流(附件复制优先使用)

1.1普通字节输入输出流

字节输入流

//创建输入流
FileInputStream fis = new FileInputStream("C:\\io_homework\\hello.txt");
//一次读一个字节数据
int byteValue = fis.read();
//一次读1024长度的内容,返回读取到数据的长度
byte[] bytes = new byte[1024];
int length = fis.read(bytes);
//关闭输入流
fis.close();

字节输出流

//创建输出流
FileOutputStream fos = new FileOutputStream("C:\\io_homework\\world.txt");
//将指定内容写入文件
fos.write("hello world".getBytes("utf-8"));
//换行
fos.write("\r\n".getBytes());
//写指定个数个数据到文件
fos.write("hello world".getBytes("utf-8"),0,5);
//关流
fos.close();

1.2缓冲字节输入输出流

缓冲字节输入流

//创建字节缓冲输入流
BufferedInputStream bis = new BufferedInputStream(new FileInputStream("C:\\hello.txt"));
//一次读一个字节数据
int byteValue = bis.read();
//一次读1024长度的内容,返回读取到数据的长度
byte[] bytes = new byte[1024];
int length = bis.read(bytes);
//关闭输入流
bis.close();

缓冲字节输出流

//创建字节缓冲输出流
BufferedOutputStream bos = new BufferedOutputStream(new FileOutputStream("C:\\world.txt"));
//将指定内容写入文件
bos.write("hello world".getBytes("utf-8"));
//换行
bos.write("\r\n".getBytes());
//写指定个数个数据到文件
bos.write("hello world".getBytes("utf-8"),0,5);
//关流
bos.close();

1.3乱码问题

出现乱码的本质

文件本身的编码格式和我们操作文件的编码格式不一样

划重点!!!

BGK编码集,一个中文对应2个字节
UTF-8编码集,一个中文对应3个字节

所以操作过程中编码集如果不一致就会导致乱码

2.字符流(文本文件可以使用)

2.1普通字符输入输出流

字符输入流

//一次读1024字符长度的内容,返回读取到数据的长度
//创建字符输入流
FileReader reader = new FileReader("C:\\jn27_workspace\\io_project\\io_homework\\hello.txt");
//一次读一个字符数据
char charValue = (char)reader.read();
char[] chars = new char[1024];
int read = reader.read(chars);
//关闭字符输入流
reader.close();

字符输出流

//创建字符输出流
FileWriter writer = new FileWriter("C:\\jn27_workspace\\io_project\\io_homework\\hello.txt");
//将指定内容写入文件
writer.write("hello world");
//换行
writer.write("\r\n");
//写指定个数个数据到文件
writer.write("hello world",0,5);
//关流
writer.close();

2.2缓冲字符输入输出流

缓冲字符输入流

  • 原始读取内容(用char[]去接收数据)

    //1.创建字符缓冲输入流对象
BufferedReader br = new BufferedReader(new FileReader("io_homework\\bw.txt"));
//2.读数据
char[] chs = new char[1024];
int len;
while ((len=br.read(chs))!=-1) {
  System.out.print(new String(chs,0,len));
}
//3.关流
br.close();

  • 使用字符缓冲输入流特有方法读取(一读读一行)

    //1.创建字符缓冲输出流对象
BufferedReader br = new BufferedReader(new FileReader("io_homework\\bw.txt"));
//2.读数据
String line;
while ((line=br.readLine())!=null) {
    System.out.print(line);
}
//3.关流
br.close();

缓冲字符输出流

//1.创建字符缓冲输出流对象
BufferedWriter bw = new BufferedWriter(new FileWriter("io_homework\\bw.txt"));
//2.1将数据写到文件中
bw.write("hello");
//2.2换行
bw.write("\r\n");
//2.3换行 字符缓冲输出流换行更简单快捷
bw.newLine();
//2.4刷新流
bw.flush();
//3.关闭
bw.close();
  • 注意 缓冲字符输出流独有的bw.newLine()表示写入换行

3.转换流(用指定字符集读写文件)

文件在操作过程字符集必须和文件本身的字符集一模一样
如果文件时utf-8,就用utf-8读取
如果文件时gbk,就用gbk读取

3.1用utf-8格式读取文件内容

//采用指定编码从文本文件中读取内容
BufferedReader br = new BufferedReader(new InputStreamReader(new FileInputStream("C:\\a.txt"), "UTF-8"));
String line = null;
while ((line=br.readLine())!=null) {
	System.out.println(line);
}
br.close();

3.2用utf-8格式把指定内容写入文件

BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(new FileOutputStream("C:\\a.txt"), "UTF-8"));
bw.write("I am 老王");
bw.close();

4.对象流

4.1序列化和反序列化

  • 使用前提
    • 需要让实体类实现Serializable接口
序列化和反序列化概念

    序列化

    ​将对象存到文件中

    反序列化

    ​将文件读取到内存中,转换成对象

在序列化的时候不想让当前类中的某个成员变量属性被序列化,可以使用瞬态,属性加transient关键子

private transient int age;

4.2将对象写入文件

//1.创建序列化流对象
ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("io_homework\\oos.txt"));
//2.创建对象
目标类 对象 = new 目标类();
//3.将对象写入文件
oos.writeObject(对象);
oos.close();

4.3将文件中的数据读取并且转换成对象

//1.创建反序列化流对象
ObjectInputStream ois = new ObjectInputStream(new FileInputStream("io_homework\\oos.txt"));
//2. 直接读取目标对象,注意之前怎么存,现在就怎么接收
目标类 对象 = (目标类)ois.readObject();
//3.对对象进行操作
System.out.println(对象);
//4.关流
ois.close();

5.Properties(重点)

Properties本身就是一个Map,他实际应用场景是操作配置文件 xxx.properties

5.1将数据写入Properties

//1.指定properties文件存放位置
FileWriter writer = new FileWriter("io_homework\\target.properties");
//2.创建Properties对象
Properties pp = new Properties();
//3.存数据
pp.setProperty("key1","value1");
pp.setProperty("key2","value2");
//4.将数据写入target.properties
pp.store(writer,null);

5.2从配置文件中获取指定key的值

//指定要读取的target.properties配置文件
FileInputStream fis = new FileInputStream("io_homework\\target.properties");
//创建对象
Properties pp = new Properties();
//Properties加载我们获取文件的输入流
pp.load(fis);
//获取target.properties配置文件中指定名称的key的值
String value1 = pp.getProperty("key1");

5.3使用ResourceBundle获取properties对应key的值

/**
* 1.配置文件在src目录
* 2.指定配置文件的时候不需要给后缀名
*/
ResourceBundle bundle = ResourceBundle.getBundle("target");
String key2 = bundle.getString("key1");
System.out.println(key2);

/**
 * 配置文件一定要放在src目录下!!!!!
 */
public class PropertiesTestCase {
    public static void main(String[] args) throws IOException {
        method3();

    }

    private static void method3() {
        ResourceBundle bundle = ResourceBundle.getBundle("db");
        String url = bundle.getString("url");
        System.out.println(url);
    }

    private static void method2() throws IOException {
        //去编译目录下去找db.properties文件,将其转换成流
        InputStream is = PropertiesTestCase.class.getClassLoader().getResourceAsStream("db.properties");
        Properties pp = new Properties();
        pp.load(is);
        String url = pp.getProperty("url");
        System.out.println(url);
    }

    private static void method1() throws IOException {
        FileInputStream fis = new FileInputStream("demo/src/db.properties");
        Properties pp = new Properties();
        pp.load(fis);
        String url = pp.getProperty("url");
        System.out.println(url);
    }
}
